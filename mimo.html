Leitura executiva

`MiMo` subiu de patamar dentro do hub. A pagina oficial de `MiMo V2.5-Pro` (lancada 27/04/2026) entrega o suficiente para uma leitura mais profissional: ha benchmark oficial publicado, ha comparativos nominais contra referencias conhecidas e ha regras publicas de consumo do Token Plan.

Resumo curto: `MiMo-V2.5-Pro` (1.02T params, 42B active, 1M ctx) ja merece estar no estudo como provider/modelo serio e com documentacao propria robusta. O que ainda impede ela de subir para a briga principal de assinatura e uma coisa bem especifica: nao achei a tabela publica final com preco por tier + creditos por tier + franquia mensal/anual no mesmo nivel de auditabilidade de `GLM`, `Kiro` ou `Cursor`.

MiMo-V2.5-Pro — Novidades de 27/04/2026

Nova flagship da familia: 1.02T parametros totais, 42B ativos (Mixture-of-Experts), arquitetura hybrid attention com 1M tokens contexto. Open source sob licenca permissiva (pesos no Hugging Face).

Specs tecnicas

  • 1.02T params total, 42B active (MoE)
  • 1M token context window
  • Hybrid attention: SWA + GA (6:1 ratio, 128-token window)
  • Multi-Token Prediction (MTP) nativo
  • FP8 (E4M3) mixed precision
  • Treinado em 27T tokens, contexto estendido ate 1M

Agentic benchmarks

  • SysY Compiler (Rust): 672 tool calls, 4.3h, 233/233 passed
  • Video Editor: 8.192 linhas, 1.868 tool calls, 11.5h autonomas
  • FVF-LDO: otimizacao analogica multi-metrica em ~1h
  • Token efficiency: 40-60% menos tokens que Opus 4.6/Gemini 3.1 Pro/GPT-5.4
  • MiMo Coding Bench: fechando gap para Opus 4.6
Integração: V2.5-Pro compativel com `Claude Code`, `OpenCode`, `Kilo`. Acesso via API Platform ou AI Studio.

Leitura final do hub (atualizado 28/04/2026)

O que faz `MiMo` subir

  • V2.5-Pro (27/04): 1.02T params, 42B active, open source, 1M ctx.
  • Token efficiency: 40-60% menos tokens que Opus 4.6/Gemini 3.1 Pro/GPT-5.4.
  • Agentic tasks reais: SysY compiler (672 tool calls, 4.3h), Video Editor (1.868 calls, 11.5h), FVF-LDO (~1h).
  • `Token Plan` atualizado com reset de creditos (21/04) + diferenca V2.5 vs V2.5-Pro (1 vs 2 credits/token).

O que ainda segura

  • Falta tabela publica final com `preco por tier` e `creditos por tier` (nao legivel via fetch; esta em imagem na pagina oficial).
  • Benchmarks V2.5-Pro nao sao exatamente as mesmas linhas centrais da matriz do hub.
  • Para recomendacao economica principal, legibilidade operacional ainda perde para `GLM`, `Kiro` e `Cursor`.
  • Nao serve para seu perfil extremo: 240M tok/mes exigiria ~240M-480M credits; nenhum tier publico suporta.

Conclusao

  • `MiMo` deve continuar no estudo como `provider em observacao`.
  • V2.5-Pro eleva de "benchmark" para "solucao agentica real" com eficiencia de custo forte.
  • Se Xiaomi abrir pricing/quota clara por tier, pode subir rapido no ranking.
  • Comparado: GLM Coding Plan ($18-160/mes) e OpenCode Go ($10/mes) continuam sendo melhores para uso massivo.
