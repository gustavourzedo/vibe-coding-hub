Os 5 principios do vibe coding sustentavel

1. Reduza linguagem inutil

Todo texto floreado do agente vira custo recorrente. Ferramentas como `Caveman` atacam exatamente esse desperdicio.

2. Troque contexto bruto por contexto estruturado

Mapas, grafos e memoria duravel escalam melhor do que despejar o repositorio inteiro no prompt.

3. Dê olhos ao agente

Browser tools e MCPs de navegador reduzem o ciclo cego de gerar, errar, reinterpretar e gastar tokens de novo.

4. Use MCP certo para o tipo de problema

`Context7` para docs atuais, `Sequential Thinking` para planos, `Filesystem` para escopo local e `DevTools` para runtime real.

5. Verifique cedo

Quanto mais tarde voce detecta que a tese do agente estava errada, mais caro fica o retrabalho.

Caveman

Ferramenta real Multiplataforma

Como funciona

`Caveman` e uma skill/plugin de compressao de linguagem para agentes de coding. A ideia e simples: fazer o agente falar muito menos, mantendo substancia tecnica. O projeto hoje se posiciona como `~65% a 75% fewer output tokens`, com suporte a `Claude Code`, `Codex`, `Gemini CLI`, `Cursor`, `Windsurf`, `Cline`, `Copilot` e outros via `npx skills`.

Como usar em IDEs

  • `Codex`: plugin/skill e hook repo-local para auto-ativacao.
  • `Cursor`: `npx skills add JuliusBrussee/caveman -a cursor`.
  • `Claude Code`: plugin de marketplace com ativacao forte.
  • `Gemini CLI`: extensao oficial do projeto.
Quando ele ajuda mais: sessoes longas com muita explicacao desnecessaria, reviews extensos, debugging iterativo e agentes que narram demais o que estao fazendo.
Onde tomar cuidado: compressao demais pode piorar onboarding, handoff e raciocinios que pedem explicacao longa para humanos. O ideal e usar `Caveman` como modo de execucao, nao como unica linguagem do projeto.

Graphify

Correcao importante: o nome/dominio que faz sentido hoje para esta tese e `graphify.homes`. Se voce vinha chamando de `graphify.net`, vale atualizar a referencia no hub para evitar ambiguidade publica.
Hospedado + OSS Arquitetura

Como funciona

`Graphify` transforma codigo, docs, PDFs, screenshots, diagramas e transcricoes em um grafo consultavel. Em vez de so fazer chunk + retrieval, ele modela estrutura, relacoes e artefatos exportaveis como `graph.html`, `graph.json` e `GRAPH_REPORT.md`.

Como usar em IDEs

  • Hosted: subir um ZIP e explorar o grafo no navegador.
  • Open-source/local: usar a tooling do projeto para integrar com `Claude Code`, `Codex`, `OpenCode`, `Cursor` e `Gemini CLI`.
  • Melhor uso: bases grandes, legadas ou multimodais, onde keyword search ja nao basta.
Onde ele brilha no vibe coding: arquitetura, debugging de dependencia cruzada, impacto de mudanca, descoberta de "god nodes", e reducao de contexto repetido entre sessoes.

4 stacks praticas de vibe coding sustentavel

Stack economica para dia a dia

  • Assinatura: `OpenCode Go` ou `GLM Pro`.
  • Ferramentas: `Context7` + `Filesystem` + `Caveman`.
  • Boa para: CRUD, backend, scripts, refactors curtos.

Stack web app com verificacao real

  • Assinatura: `GLM`, `Codex` ou `Cursor`.
  • Ferramentas: `Context7` + `Chrome DevTools MCP` + `Sequential Thinking`.
  • Boa para: front-end, layout, forms, performance e bugs que so aparecem no runtime.

Stack para repositorio grande e confuso

  • Assinatura: qualquer uma que sustente sessao longa.
  • Ferramentas: `Graphify` + `Filesystem` + `Context7`.
  • Boa para: legado, monorepo, onboarding e arqueologia arquitetural.

Stack de frontier open-model

  • Assinatura: `Ollama Max` ou stack equivalente open-frontier.
  • Ferramentas: `Context7` + `Graphify` + navegador so quando necessario.
  • Boa para: autonomia sobre `GLM`, `Kimi`, `Qwen` e workflows menos dependentes de vendor fechado.

Regras praticas para gastar menos

  • Prefira pedir plano antes de patch grande em tarefas ambíguas.
  • Evite mandar o repositorio inteiro quando um grafo, mapa ou resumo local resolve.
  • Use `Context7` cedo quando a tarefa depende de framework ou API viva.
  • Use browser MCP para confirmar bug visual antes de abrir loops longos de tentativa e erro.
  • Comprima saidas recorrentes e handoffs com `Caveman`, mas preserve um modo normal para docs e decisoes humanas.
  • Transforme aprendizados persistentes em memoria, regras ou artefatos exportaveis em vez de repetir o contexto na unha.
