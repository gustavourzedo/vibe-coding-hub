Modelo mental do stack

1. Modelo

Opus, GPT, Gemini, GLM, Kimi, DeepSeek, MiniMax. Define inteligencia bruta, contexto e custo por token.

2. Assinatura / provider

Claude Max, ChatGPT, GLM Coding Plan, OpenCode Go, Google AI. Define franquia, teto, throttle e previsibilidade.

3. IDE / harness

Codex, Cursor, Claude Code, Antigravity, OpenCode, Windsurf. Define como o modelo mexe no codigo, terminal, git e contexto.

4. Orquestrador

n8n, LangGraph, Mastra, CrewAI, Dify, Temporal. Define como automatizar rotinas, coordenar agentes e transformar o estudo em sistema vivo.

Leitura curta: para vibe coding direto, a decisao principal continua sendo assinatura + harness. Orquestradores ajudam mais quando voce quer repetir processos, criar pipelines, auditar uso, acionar varios agentes ou transformar o hub em um produto/operacao continua.

Casos de uso concretos para este estudo

1

Snapshot semanal

Ler fontes locais permitidas, atualizar `telemetry/latest_telemetry.json` e gerar resumo de uso por ferramenta.

2

Watch de preco

Checar paginas oficiais de pricing e marcar diferenca de preco, quota, modelo ou restricao operacional.

3

Fila de pesquisa

Separar fontes novas em "oficial", "comunidade" e "rumor", reduzindo risco de atualizar o hub com dado fraco.

4

Relatorio de decisao

Gerar uma leitura semanal: capacidade real usada, gargalos, plano mais economico e plano de contingencia.

5

Produto futuro

Transformar a documentacao viva em app com historico, alertas, scoring versionado e comparativos por perfil.

Melhor primeira implementacao: n8n para automacao externa + scripts locais versionados no repo. Se o hub evoluir para produto proprio, migrar o nucleo inteligente para LangGraph ou Mastra, mantendo n8n apenas como camada de integracao/notificacao.

Como escolher por situacao

Quero monitorar meu uso real

Escolha: n8n + scripts locais.

  • Executa rotina semanal.
  • Integra email, webhook, planilha, GitHub e notificacoes.
  • Nao precisa reinventar painel de automacao.

Quero criar agentes do produto

Escolha: LangGraph ou Mastra.

  • Melhor controle de estado, memoria e grafo.
  • Mais testavel e versionavel em codigo.
  • Permite avaliacoes e roteamento por modelo.

Quero montar prototipo visual

Escolha: Dify, Flowise ou CrewAI AMP.

  • Menos codigo inicial.
  • Bom para demos e RAG interno.
  • Menos flexivel para engenharia fina.

Quero confiabilidade industrial

Escolha: Temporal por baixo.

  • Retries, estado duravel e processos longos.
  • Otimo para produto serio em escala.
  • Exagerado para o hub nesta fase.

Quero viver no ecossistema OpenAI

Escolha: OpenAI Agents SDK.

  • Tools, handoffs, guardrails e tracing.
  • Combina bem com Codex/OpenAI stack.
  • Menos neutro que LangGraph/Mastra.

Quero integrar SaaS rapido

Escolha: Make, Zapier ou Pipedream.

  • Melhor para acionar apps externos.
  • Pricing por task/operacao/credito exige cuidado.
  • Bom complemento, nao base principal do estudo.

Ferramentas em observacao

Ferramenta Por que observar Risco / ressalva Decisao atual
Flowise Visual builder popular para agentes, chatflows e LLM apps. Mais forte para app/RAG visual do que para telemetria local. Revalidar pricing e postura de seguranca antes de self-host publico. Observacao, nao primeira escolha.
Langflow Visual builder conhecido, util para prototipos RAG. DataStax Langflow cloud foi marcado como deprecated em docs oficiais; usar com cautela e preferir OSS/local se fizer sentido. Observacao tecnica.
Zapier Agents Boa superficie para agentes que executam acoes em SaaS. Task-based pricing pode ficar caro quando agentes executam muitas acoes. Complementar para integracoes externas.
Pipedream Developer-friendly para APIs, webhooks e workflows serverless. Credit-based por compute; menos "painel de operacao" que n8n para este caso. Alternativa se o hub virar API/integracoes dev-heavy.

Recomendacao para o hub agora

Fase 1: manter o frontend estatico e adicionar uma rotina simples de telemetria. O melhor encaixe pratico e `n8n` orquestrando tarefas externas + scripts versionados aqui no repo.

Agora

n8n para agenda, coleta, notificacao e abertura de tarefas de atualizacao.

Se virar app

Mastra se quisermos TypeScript-first; LangGraph se quisermos grafo robusto, estados e avaliacao profunda.

Evitar por enquanto

Temporal como primeira camada e Zapier/Make para loops intensivos de IA, porque podem aumentar custo/complexidade cedo demais.
