O gate de Tier

Regra do hub daqui para frente: se a ferramenta nao permite usar a sua propria assinatura, conta ou chave de API, ela no maximo entra como Tier 2. Pode continuar relevante, pode continuar boa, mas deixa de ser superficie principal para quem quer vibe coding sustentavel, flexivel e economico.

O que conta como BYOK

API key direta, conta propria, provider custom, suporte nativo a modelos externos ou ate um fluxo parcial/limitado que deixe voce usar sua stack privada.

O que nao conta

Somente usar os modelos do proprio produto sem caminho serio de trazer sua conta. Nesses casos a ferramenta pode ser otima, mas vira no maximo camada secundaria.

Por que isso importa

Sem BYOK voce perde autonomia quando a quota aperta, quando quer trocar de frontier ou quando precisa aproveitar assinatura paralela como `GLM`, `OpenCode Go` ou `Claude`.

Destaque novo: por que o Zed entrou tao forte

O que ele acertou para este hub

  • Tem AI hospedada, mas nao obriga assinatura fechada para tudo.
  • Tem suporte a `custom providers`, `own API keys` e configuracao de modelos.
  • Tem `Agent Panel`, ferramentas e permissoes para trabalho agentico mais serio.
  • Tem `external agents` via `ACP`, o que amplia bastante a superficie futura.
  • Tem camada free real com `Zeta` para nao te deixar totalmente travado.

O que ainda exige cautela

  • Parte da experiencia ainda depende de como cada provider externo e configurado.
  • `ACP` e agentes externos sao poderosos, mas nem sempre tao polidos quanto o fluxo nativo.
  • Para algumas pessoas ele ainda e menos popular no ecossistema que `Cursor` ou `VS Code`, o que reduz a massa de tutoriais e receitas prontas.
Leitura do hub: se o criterio mais importante passa a ser autonomia de stack sem abrir mao de UX moderna, `Zed` deixa de ser "nome que faltou" e passa a ser candidato serio ao topo.

Leituras praticas por superficie

Melhor superficie principal hoje

Zed

Se voce quer uma resposta unica e atual para "qual superficie mais preparada para vibe coding sustentavel com liberdade de stack?", o `Zed` ficou na frente nesta rodada.

Melhor command center

Codex

Se a prioridade e multiagente, fluxo profissional de terminal, browser e integracao com a stack OpenAI, `Codex` continua fortissimo.

Melhor host universal

VS Code

Como host, `VS Code` segue sendo a base mais flexivel para plugar quase qualquer estrategia de provider e harness.

Se voce ama o Cursor

Ele continua muito viavel. A diferenca e que agora a documentacao deixa claro: ele nao pode mais vencer "por magia de UX" ignorando o criterio de autonomia de stack.

Se voce quer stack chinesa

`OpenCode`, `VS Code host` e `Zed` tendem a encaixar melhor quando a meta e usar `GLM`, `Kimi`, `Qwen`, `DeepSeek` e rotas alternativas com menos atrito.

Se voce quer fallback quando a quota seca

`Zed` e `VS Code host` ajudam mais porque aceitam composicao mais flexivel entre free, barato e premium sem te prender a um jardim fechado.
